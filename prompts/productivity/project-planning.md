# Project Planning Prompts

## Overview

These AI prompts help you plan projects effectively from conception to execution. Use them to define objectives, create timelines, identify resources, anticipate risks, and establish success metrics for any type of project.

## Project Initiation

### Project Charter Creation

**Prompt**: "Help me create a comprehensive project charter for [PROJECT NAME]:

**Initial concept:**
[Brief description]

**Known goals:**
[What you want to achieve]

Provide a structured charter including:
1. **Project Vision**: Clear, inspiring statement
2. **Objectives**: 3-5 SMART goals
3. **Scope**: What's included and excluded
4. **Key Deliverables**: Major outputs
5. **Stakeholders**: Who's involved and their roles
6. **Success Criteria**: How we'll measure success
7. **Constraints**: Budget, time, resources
8. **Assumptions**: What we're taking for granted
9. **Risks**: Initial concerns
10. **High-Level Timeline**: Major phases"

### Problem Definition

**Prompt**: "Analyze this project need and help define the problem we're solving:

**Situation:**
[Current state]

**Desired outcome:**
[Future state]

**Challenges:**
[Known obstacles]

Provide:
- Clear problem statement
- Root cause analysis
- Impact if unsolved
- Success definition
- Project justification
- Alignment with goals
- Alternative approaches"

## Project Planning

### Comprehensive Project Plan

**Prompt**: "Create a detailed project plan for [PROJECT]:

**Project details:**
- Goal: [Objective]
- Duration: [Timeframe]
- Budget: [Amount]
- Team size: [Number]

Provide a complete plan with:

**Phase 1: Planning**
- Requirements gathering
- Stakeholder alignment
- Resource allocation

**Phase 2: Design/Development**
- Key activities
- Milestones
- Dependencies

**Phase 3: Testing/Validation**
- Quality checks
- Feedback loops

**Phase 4: Launch/Implementation**
- Rollout strategy
- Success metrics
- Monitoring plan

**Phase 5: Closure**
- Handoff procedures
- Documentation
- Lessons learned

Include timeline, resources, and risks for each phase."

### Milestone Definition

**Prompt**: "Define project milestones for [PROJECT] with [DURATION]:

Create 5-8 major milestones that:
- Mark significant progress
- Are measurable and specific
- Have clear success criteria
- Build toward final delivery

For each milestone, provide:
- Name and description
- Target completion date
- Dependencies (what must finish first)
- Deliverables
- Success criteria
- Risk factors
- Celebration/communication plan"

### Resource Planning

**Prompt**: "Plan resources needed for [PROJECT]:

**Project scope:**
[Description]

**Timeline:**
[Duration]

**Budget:**
[Available budget]

Identify resource requirements:

**Human Resources:**
- Roles needed
- Skills required
- Time commitment
- Internal vs external

**Technology/Tools:**
- Software platforms
- Hardware needs
- Infrastructure

**Budget Allocation:**
- Personnel costs
- Technology expenses
- Operational costs
- Contingency (10-20%)

**Material Resources:**
- Physical materials
- Facilities
- Equipment

Provide cost estimates and prioritization."

## Risk Management

### Risk Assessment

**Prompt**: "Conduct a risk assessment for [PROJECT]:

**Project context:**
[Description]

Identify and analyze risks:

**High Priority Risks:**
- Description
- Probability (High/Med/Low)
- Impact (High/Med/Low)
- Mitigation strategy
- Contingency plan
- Risk owner

**Medium Priority Risks:**
[Same structure]

**Low Priority Risks:**
[Same structure]

Create a risk matrix and recommend:
- Preventive actions
- Monitoring approach
- Escalation procedures
- Buffer time/budget needed"

### Dependency Mapping

**Prompt**: "Map dependencies for [PROJECT]:

**Major tasks/deliverables:**
[List key items]

Identify:
- **Sequential dependencies**: What must happen in order
- **Parallel tracks**: What can happen simultaneously
- **External dependencies**: What we rely on from others
- **Critical path**: The longest dependent path
- **Bottlenecks**: Where delays impact everything
- **Quick wins**: Independent items we can complete early

Provide a dependency chart and highlight:
- Tasks that can start immediately
- Tasks waiting on dependencies
- Most critical dependencies to monitor"

## Timeline Creation

### Project Timeline

**Prompt**: "Create a project timeline for [PROJECT] from [START DATE] to [END DATE]:

**Major deliverables:**
[List items]

**Team capacity:**
[Available resources]

**Constraints:**
[Fixed dates, dependencies]

Provide:
1. **Week-by-week breakdown** (for projects <3 months)
2. **Month-by-month plan** (for projects >3 months)
3. **Key milestones** with dates
4. **Buffer time** between critical phases
5. **Review/approval points**
6. **Go/no-go decision points**
7. **Launch preparation period**

Highlight:
- Critical deadlines
- Resource-intensive periods
- Risk windows
- Slack time for adjustments"

### Sprint Planning

**Prompt**: "Plan [NUMBER] sprints for [PROJECT]:

**Sprint duration:**
[Typically 1-4 weeks]

**Team capacity:**
[Story points or hours per sprint]

**Product backlog:**
[Prioritized list]

For each sprint, define:
- Sprint goal
- Selected user stories/tasks
- Acceptance criteria
- Definition of done
- Sprint deliverables
- Demo plan
- Retrospective focus

Ensure each sprint delivers value and builds toward project goal."

## Stakeholder Management

### Stakeholder Analysis

**Prompt**: "Analyze stakeholders for [PROJECT]:

Identify all stakeholders and categorize:

**Power/Interest Matrix:**

**High Power, High Interest** (Manage Closely)
- Who they are
- Their interests
- Communication frequency
- Influence strategy

**High Power, Low Interest** (Keep Satisfied)
[Same structure]

**Low Power, High Interest** (Keep Informed)
[Same structure]

**Low Power, Low Interest** (Monitor)
[Same structure]

Provide:
- Communication plan for each group
- Key messages
- Engagement strategy
- Risk mitigation for opposition"

### Communication Plan

**Prompt**: "Create a project communication plan for [PROJECT]:

**Stakeholders:**
[List groups]

**Project duration:**
[Timeline]

Define communication strategy:

**For each stakeholder group:**
- What information they need
- How often to communicate
- Best communication channels
- Format (email, meeting, dashboard)
- Who is responsible
- Escalation process

**Communication types:**
- Regular status updates
- Milestone announcements
- Risk alerts
- Change notifications
- Success celebrations

Include templates for key communications."

## Success Metrics

### KPI Definition

**Prompt**: "Define KPIs for [PROJECT]:

**Project goals:**
[Objectives]

Create a balanced scorecard with:

**Outcome Metrics** (Did we achieve the goal?)
- Primary success indicators
- Target values
- Measurement method

**Output Metrics** (Did we deliver what we promised?)
- Deliverable completion
- Quality standards
- Timeline adherence

**Process Metrics** (Are we working efficiently?)
- Team velocity
- Resource utilization
- Budget tracking

**Impact Metrics** (What's the business value?)
- ROI calculation
- Benefit realization
- User adoption

For each KPI, specify:
- Measurement frequency
- Responsible party
- Target vs actual tracking
- Corrective action triggers"

## Best Practices

1. **Start with why** - Clear purpose drives everything
2. **Plan for change** - Build flexibility into plans
3. **Define success early** - Know what done looks like
4. **Identify constraints** - Work within reality
5. **Map dependencies** - Understand what depends on what
6. **Build in buffers** - Things take longer than expected
7. **Engage stakeholders** - Alignment prevents problems
8. **Document assumptions** - Make the implicit explicit
9. **Plan for risks** - Anticipate and mitigate
10. **Keep it simple** - Complex plans fail
